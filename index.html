Aufgabe 1.1. Fachliche Argumentation über Erfolgsprinzipien des WWW

Mit welchen fachlichen Argumenten wurde das WWW-Proposal von TBL abgelehnt?
  • TBL hat die Wagheit des Konzepts nicht verstanden (er verwechselt es mit der Evolvierbarkeit)
  • Die Hypertext-Community hatte den Eindruck, dass sie schon mehr erreicht hatten als TBL (insbesondere wegen 'broken Links')
Was sind die fachlichen Argumente, warum das WWW dennoch ein Erfolg wurde?
  • Die Unterteilung in die verschiedenen orthogonalen Technologien:
    • HTTP
    • HTML
    • eindeutige Identifikation (URI)
Was wäre der Preis für die garantierte Verhinderung von “broken links“?
  • zentrale Geschlossenheit
  • Kontrolle

Aufgabe 1.2. HTTP

Sie bekommen im Browser den HTTP Status Code 200. Was bedeutet das?
  • "alles in Ordnung"
  • Anfrage erfolgreich verlaufen
Sie bekommen im Browser den HTTP Status Code 301. Was hat das zu bedeuten?
  • Seite existiert an der angegebenen Stelle nicht mehr
  • User wird zur "richtigen" Seite weitergeleitet
Sie bekommen im Browser den HTTP Status Code 400. Was hat das zu bedeuten? Was können Sie dagegen tun?
  • Anfrage kann nicht verarbeitet werden (client-seitiger Fehler)
  • User sollte URL überprüfen, gegebenenfalls anpassen und erneut versuchen
Sie bekommen im Browser den HTTP Status Code 403. Was hat das zu bedeuten? Was können Sie dagegen tun?
  • Server hat die Anfrage zwar verstanden, lässt den Client aber nicht zugreifen
  • Client sollte seine Berechtigungen für diese Seite überprüfen
In einer Webanwendung benötigen Sie eine OPTIONS-Anfrage, die die Optionen des Servers vor dem eigentlichen Zugriff erfragen soll. Aus Performanzgründen soll die Abfrage jedoch cacheable sein. Wie könnten Sie dafür eine Lösung angehen?
  • über eine zusätzliche GET Anfrage oder auch nur eine HEAD Anfrage und die gewünschten Informationen dort unterbringen
